# 子音対応・テキスト読み上げ 要件定義書

## 1. 概要

### 1.1 目的

Phase 1-5 で構築した「声の楽器」（母音合成 + 自動歌唱）を拡張し、**全ひらがな音素の発声**と**テキスト読み上げ**を実現する。

### 1.2 達成目標

- 日本語の全ひらがな（清音・濁音・半濁音・拗音・特殊拍）が物理モデルベースで発声できる
- ひらがなテキストを入力すると、音素分解→声道制御→発声が自動で行われる
- 簡易韻律（ピッチアクセント・文末下降）により自然な読み上げを実現する

### 1.3 技術的前提

- 既存の44区間Kelly-Lochbaumモデル、KLGLOTT88/LF声門音源を基盤とする
- AudioWorklet の process() 内 GC-free 制約を維持する
- SharedArrayBuffer 不使用（GitHub Pages 互換）を維持する

## 2. 声道モデル拡張の機能要求

### 2.1 子音の声道断面積制御

**対応音素一覧**

| 音素 | IPA | カテゴリ | 閉鎖/狭窄位置(16点idx) | 面積(cm²) |
|------|-----|---------|----------------------|-----------|
| か/が | /k/, /g/ | 破裂(軟口蓋) | 5-6 | 0.05 |
| た/だ | /t/, /d/ | 破裂(歯茎) | 2-3 | 0.05 |
| ぱ/ば | /p/, /b/ | 破裂(両唇) | 0-1 | 0.05 |
| さ | /s/ | 摩擦(歯茎) | 2-3 | 0.10-0.15 |
| し | /ɕ/ | 摩擦(歯茎硬口蓋) | 3-4 | 0.10-0.20 |
| は | /h/ | 摩擦(声門) | 声門ノイズ | -- |
| ひ | /ç/ | 摩擦(硬口蓋) | 4-5 | 0.15-0.30 |
| ふ | /ɸ/ | 摩擦(両唇) | 0-1 | 0.15-0.30 |
| ざ | /z/, /ʑ/ | 摩擦(有声) | 2-4 | 0.10-0.25 |
| ち | /tɕ/ | 破擦 | 閉鎖3-5→摩擦2-4 | 0.05→0.15 |
| つ | /ts/ | 破擦 | 閉鎖1-3→摩擦1-2 | 0.05→0.12 |
| じ/ぢ | /dʑ/ | 破擦(有声) | 閉鎖3-5→摩擦2-4 | 0.05→0.15 |
| ず/づ | /dz/ | 破擦(有声) | 閉鎖1-3→摩擦1-2 | 0.05→0.12 |
| ま | /m/ | 鼻(両唇) | 0-1 閉鎖+鼻腔開 | 0.05(口腔) |
| な | /n/ | 鼻(歯茎) | 2-3 閉鎖+鼻腔開 | 0.05(口腔) |
| に | /ɲ/ | 鼻(硬口蓋) | 3-4 閉鎖+鼻腔開 | 0.05(口腔) |
| ら | /ɾ/ | 弾き(歯茎) | 2-3 瞬間接触 | 0.05(5-15ms) |
| や | /j/ | 半母音 | /i/形状から遷移 | 0.30 |
| わ | /w/ | 半母音 | /u/形状から遷移 | 0.30 |

**MIN_AREA 二段階制限**

| レイヤー | 下限(cm²) | 用途 |
|---------|-----------|------|
| UI ドラッグ | 0.3 | ユーザ操作時の安全下限（現行値維持） |
| プログラム制御 | 0.01 | 子音の閉鎖・狭窄を模擬（r=0.995、数値安定） |

**コアーティキュレーション**: 子音の閉鎖/狭窄区間のみプリセット値を適用し、残り区間は先行母音から後続母音への線形補間とする。

### 2.2 狭窄ノイズ注入

VocalTract の散乱ループ内で、狭窄区間 k の前進波にバンドパスノイズを加算。

```
f[k] += frictionGain * bandpassNoise
```

**スペクトル整形パラメータ（Biquad BPF）**

| 音素 | 中心周波数(Hz) | 帯域幅(Hz) | frictionGain |
|------|---------------|-----------|-------------|
| /s/ | 6000 | 4000 | 0.6-0.8 |
| /ɕ/ | 3750 | 2500 | 0.5-0.7 |
| /ç/ | 3500 | 3000 | 0.3-0.5 |
| /ɸ/ | 2500 | 3000 | 0.2-0.4 |
| /h/ | 声道共鳴依存 | -- | 0.3-0.5 |
| バースト | 4000 | 6000 | 0.4-0.6 |

GC-free: LCG(2ops) + Biquad(5ops) + 加算(1op) = **~8 ops/sample**。

### 2.3 鼻腔分岐管

| パラメータ | 値 |
|-----------|-----|
| 区間数 | 30（長さ約12cm、固定断面積、ハードコード） |
| 鼻孔放射 | 口腔放射と同等のハイパスフィルタ |
| 3ポート接合 | `k = (A_p*f_p + A_o*b_o + A_n*b_n) / A_sum`（乗算3回） |

**口蓋帆（velum）制御**

| 状態 | velopharyngealArea(cm²) |
|------|------------------------|
| 閉鎖（母音・口腔子音） | 0.0（鼻腔計算スキップ） |
| 全開（鼻音） | 1.5-2.0 |

**撥音「ん」の異音規則**

| 後続音素 | 異音 | 口腔閉鎖位置 |
|---------|------|-------------|
| ま行 | [m] | idx 0-1 |
| な行 | [n] | idx 2-3 |
| か行 | [ŋ] | idx 5-6 |
| 語末/母音前 | [ɴ] | 口蓋帆全開、口腔閉鎖なし |

### 2.4 Worklet側サンプル精度補間

子音の5-20ms遷移はrAF(16ms)では不足。Worklet側で線形補間を実行。

```typescript
{ type: 'scheduleTransition'; targetAreas: Float64Array; durationSamples: number }
```

遷移中に新メッセージ受信時は現在値を起点に新遷移へ即座切替。

### 2.5 パフォーマンス要件

| 指標 | 上限 |
|------|------|
| 口腔管散乱 (44区間) | 129 ops/sample（現行同一） |
| 鼻腔管散乱 (30区間) | +87 ops/sample |
| 3ポート接合 | +6 ops/sample |
| ノイズ注入 | +8 ops/sample |
| 合計追加 | 現行比 **+25%以下** |

velum閉鎖時は鼻腔管スキップでコスト0。

## 3. テキスト→音声変換パイプライン

### 3.1 ひらがな→音素変換

テーブル駆動、最長一致（2文字優先→1文字fallback）。約110エントリ。

| 区分 | 文字数 | 例 |
|------|--------|-----|
| 清音 | 46字 | あ〜ん |
| 濁音 | 20字 | が〜ど |
| 半濁音 | 5字 | ぱ〜ぽ |
| 拗音 | 36組 | きゃ、しゅ、ちょ 等 |
| 特殊拍 | 3字 | っ、ー、ん |

**変換例**:
```
"さくら"   → /s,a,k,u,ɾ,a/
"しんぶん" → /ɕ,i,m,b,u,ɴ/
"ちょっと" → /tɕ,o,Q,t,o/
"にゃーん" → /ɲ,a,a,ɴ/
```

### 3.2 音素の持続時間モデル

| カテゴリ | 持続時間(ms) |
|---------|-------------|
| 母音 | 80-120 |
| 破裂子音 | 20-40 (+閉鎖60) |
| 摩擦子音 | 60-80 |
| 破擦子音 | 90 |
| 鼻子音 | 40-60 |
| 弾音 /ɾ/ | 20-30 |
| 半母音 | 30-50 |
| 促音 | 80-120 (無音) |
| ポーズ（句点） | 300-500 |
| ポーズ（読点） | 150-250 |

発話速度係数: 0.5x〜2.0x（UIスライダー）

**文位置修正**: 文頭=x1.2延伸、文末=x1.5延伸、読点前=x1.3+200msポーズ

### 3.3 韻律（F0・強度）モデル

**F0制御（ルールベース、男声デフォルト）**

| パターン | F0値 |
|---------|------|
| 平板型: 1モーラ目 Low | 110Hz |
| 平板型: 2モーラ目以降 High | 140Hz |
| 文末下降（最終2-3モーラ） | 140→80Hz |
| 疑問文（「？」検出） | +50Hz上昇 |
| Declination | `F0_base *= exp(-0.3 * t)` |

**強度テーブル**

| 音素カテゴリ | 相対振幅 | 音源タイプ |
|-------------|---------|-----------|
| 母音 /a/ | 1.0 | voiced |
| 母音 /e,o/ | 0.85 | voiced |
| 母音 /i,u/ | 0.7 | voiced |
| 鼻音 | 0.55 | voiced+nasal |
| 摩擦音 | 0.3 | noise |
| 破裂バースト | 0.4 | noise |
| 閉鎖区間 | 0.0 | silence |

### 3.4 音素タイムライン

```typescript
interface PhonemeEvent {
  phoneme: string;          // IPA表記
  startTime: number;        // 秒（文頭からの絶対時刻）
  duration: number;         // 秒
  tractAreas: number[];     // 16制御点の声道形状
  f0Start: number;          // 区間開始F0 (Hz)
  f0End: number;            // 区間終了F0 (Hz)
  sourceType: 'voiced' | 'noise' | 'voiced+noise' | 'silence';
  amplitude: number;        // 相対振幅 (0.0-1.0)
  nasalCoupling: number;    // 鼻腔結合面積 (0=閉鎖, 1.5-2.0=開放)
  constrictionNoise?: { position: number; intensity: number };
}
```

再生制御: 開始/一時停止/停止/シーク。AudioContext.currentTime 基準クロック。

## 4. UI要件

### 4.1 テキスト入力UI

| 要素 | 仕様 |
|------|------|
| textarea | `#text-input`、2行、横幅は既存UIと同一、placeholder「ひらがなを入力...」 |
| 再生ボタン | `#text-read-btn`、トグル式（読み上げ中は「停止」）、AppState=running 以外は disabled |
| 速度スライダー | `#speech-rate-slider`、range 0.5-2.0、step 0.1、default 1.0 |

### 4.2 音素タイムライン表示

- textarea 下部に横帯、音素を持続時間比例の幅で表示
- 再生中の音素をハイライト
- 声道エディタ上に子音の狭窄位置マーカーを表示

### 4.3 操作モードの排他制御

```typescript
type OperationMode = 'manual' | 'autoSing' | 'textRead';
```

| UI要素 | manual | autoSing | textRead |
|--------|--------|----------|----------|
| 声道ドラッグ | 有効 | 無効 | 無効 |
| 母音プリセット | 有効 | 無効 | 無効 |
| F0/Vol/Rdスライダー | 有効 | F0は基準値合算 | 無効 |
| Auto Singボタン | 有効 | 有効(停止用) | 無効 |
| テキスト再生ボタン | 有効 | 無効 | 有効(停止用) |

## 5. 実装フェーズ計画

| Phase | 目標 | 完了条件 | エージェント数 |
|-------|------|---------|-------------|
| **6** | 子音基盤（摩擦音・破裂音） | /s/, /k/, /t/, /p/ を手動トリガーで発声、スペクトルで確認 | 3名 |
| **7** | 鼻腔管モデル・鼻音 | /m/, /n/ が鳴る、反共鳴がスペクトルで視認 | 3名 |
| **8** | テキスト→音素→発声 | `play("こんにちは")` で5音素連続発声 | 4名 |
| **9** | テキスト読み上げUI | UIからひらがな入力→再生が動作 | 4名 |

### Phase間の依存関係

```
Phase 6 (子音基盤)
  → Phase 7 (鼻腔管)
    → Phase 8 (テキスト→音素→発声)
      → Phase 9 (UI完成)
```

## 6. 非機能要件

| カテゴリ | 要件 |
|---------|------|
| process() 予算 | 鼻腔管追加後も quantum 50%以下（~1.45ms） |
| メインスレッド | Canvas描画+タイムライン更新が 16ms(60fps)以内 |
| ブラウザ互換性 | Chrome 90+, Firefox 90+, Edge 90+ |
| 追加バンドルサイズ | Phase 6-9 合計 < 30KB (gzip後) |
| 既存非退行 | 母音プリセット・Auto Sing・スペクトル表示が正常動作 |
| レイテンシ | テキスト再生開始→発声 < 50ms |
