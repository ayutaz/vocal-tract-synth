## 1. タスク目的とゴール

Phase 1 で構築した基本パイプライン（パルス音源 + 44区間Kelly-Lochbaum + 差分放射フィルタ）を高品質化し、**5母音（あいうえお）が識別できるレベル**の音声合成を実現する。

### 達成基準

- KLGLOTT88声門音源により、自然なグロッタルパルスが生成される
- 壁面損失の導入で金属的な不自然さが解消される
- 改良放射フィルタにより、スペクトル傾斜が自然になる
- 5母音プリセットボタンを押すと、それぞれの母音が聞き分けられる音が出る
- プリセット間の遷移が滑らかで、不自然なクリックやグリッチが発生しない
- 有声/無声の切替が可能

### フォルマント目標値（成人男性）

| 母音 | F1 (Hz) | F2 (Hz) | F3 (Hz) |
|------|---------|---------|---------|
| /a/  | 800     | 1300    | 2500    |
| /i/  | 300     | 2300    | 3000    |
| /u/  | 350     | 1400    | 2500    |
| /e/  | 500     | 1900    | 2600    |
| /o/  | 500     | 800     | 2500    |

## 2. 実装する内容の詳細

### 2.1 KLGLOTT88声門音源モデル

**対象ファイル**: `src/models/glottal-source.ts`, `src/audio/worklet-processor.ts`

Phase 1 の単純パルス音源を KLGLOTT88 多項式モデルに置き換える。

#### 位相アキュムレータ方式

```
phase += F0 / sampleRate   (毎サンプル)
if phase >= 1.0: phase -= 1.0
output = klglott88(phase, OQ)
```

- 位相は 0.0 ~ 1.0 の範囲で正規化
- F0 の変更に対してサンプル単位で追従可能
- 位相リセット時は `phase -= 1.0` で端数を保持し、ドリフトを防止

#### KLGLOTT88 波形関数

多項式ベースの声門波形モデル。暗黙方程式が不要で、1サンプルあたり 4-5 ops で計算可能。

- 開放相（0 <= phase < OQ）: 多項式による滑らかな開放波形
- 閉鎖相（OQ <= phase < 1.0）: 出力 = 0

#### OQ（Open Quotient）パラメータ

- 範囲: 0.3 ~ 0.8（典型値: 0.5 ~ 0.7）
- OQ が大きい = 声門が長く開いている = ブレシーな声質
- OQ が小さい = 声門が短く開いている = 圧迫された声質
- 初期値: 0.6

### 2.2 壁面損失の実装

**対象ファイル**: `src/audio/worklet-processor.ts`, `src/models/vocal-tract.ts`

Kelly-Lochbaum の各区間に減衰係数 mu を適用する。

```
f_k[n] *= mu
b_k[n] *= mu
```

- mu = 0.999（典型値）
- 全44区間に均一に適用
- 効果: 無損失時の金属的な音が減衰し、自然な声質に近づく
- process() 内の散乱計算ループに組み込む（追加コスト: 乗算 2回/区間）

### 2.3 改良放射フィルタ

**対象ファイル**: `src/audio/worklet-processor.ts`

Phase 1 の単純差分フィルタを 1次IIRフィルタに改良する。

#### 唇端反射

```
b_0[n] = R_L * f_0[n]  (唇端での反射波)
```

- R_L = -0.75 ~ -0.9（典型初期値: -0.8）
- 負号は開放端反射（位相反転）を表す

#### 放射フィルタ

```
output[n] = f_0[n] - alpha * f_0[n-1]
```

- alpha = 0.9 ~ 0.97（典型初期値: 0.96）
- 1次差分フィルタによる +6dB/oct のスペクトル傾斜補正
- 前サンプル値 `f_0_prev` を状態変数として保持

### 2.4 声門端反射の改良

**対象ファイル**: `src/audio/worklet-processor.ts`

声門端の境界条件を改良し、より自然な反射特性にする。

```
f_{N-1}[n] = glottal_source[n] + r_glottis * b_{N-1}[n]
```

- r_glottis = 0.9 ~ 0.99（典型初期値: 0.95）
- 声門の部分的な反射を考慮
- 完全閉鎖（r=1.0）と完全開放（r=0.0）の中間

**対象ファイル**: `src/ui/controls.ts`

6つのプリセットボタンを配置する:


### 2.7 有声/無声切替

**対象ファイル**: `src/audio/worklet-processor.ts`, `src/ui/controls.ts`

```
output = AV * glottal_pulse(phase) + AN * noise()
```

- [Noise] ボタンでトグル切替
- 有声 → 無声、無声 → 有声の切替時は 5~10ms のクロスフェード
- クロスフェードはサンプル単位の線形補間で実装
- ノイズ源: ホワイトノイズ（Math.random() * 2 - 1、ただし process() 内では事前確保バッファを使用）

<!-- REVIEW: アーキテクチャレビューにて修正 — ノイズ生成でMath.random()をprocess()内で直接呼ぶことの注意点を追記。 -->
> **注意**: `Math.random()` は AudioWorklet スレッドでも利用可能だが、一部ブラウザ/バージョンで GC を誘発する可能性がある。代替として線形合同法（LCG）による自前乱数生成器を用意し、状態を `Uint32Array(1)` で管理する方式が安全。例: `seed = (seed * 1664525 + 1013904223) >>> 0; return seed / 4294967296 * 2 - 1;`

### 2.8 コサイン補間による母音遷移

**対象ファイル**: `src/models/vowel-presets.ts`, `src/ui/controls.ts`

プリセット切替時に断面積が瞬間的に変わるとクリックノイズが発生するため、滑らかな補間遷移を実装する。

- 補間方式: コサイン補間（ease-in/ease-out）
  ```
  t_smooth = 0.5 * (1 - cos(pi * t))  // t: 0→1 の線形進行
  area[i] = (1 - t_smooth) * start[i] + t_smooth * target[i]
  ```
- 遷移時間: 80~200ms（設定可能、デフォルト 150ms）
- 遷移中は毎フレーム（requestAnimationFrame）で断面積を更新し、postMessage で Worklet に送信
- 遷移中に別のプリセットが選択された場合、現在の中間状態から新しいターゲットへ遷移を開始

### 4.3 ユニットテスト

<!-- REVIEW: テスト戦略レビューにて修正 — OQ境界値テスト・F0範囲テスト・コサイン補間テスト・パフォーマンステストを追加、成功基準トレーサビリティを付与 -->

#### KLGLOTT88 波形の連続性

- 位相 0.0 ~ 1.0 の範囲で波形値が連続（不連続点がない）
- 閉鎖相（phase >= OQ）で出力が 0
- 開放相の最大値が 1.0 付近であること
- OQ を 0.3 ~ 0.8 の範囲で変化させても波形が破綻しないこと
- 位相アキュムレータが長時間実行（100万サンプル以上）でドリフトしないこと
- **境界値: OQ=0.3（最小値）で波形が正しく生成されること（開放相が非常に短い）**
- **境界値: OQ=0.8（最大値）で波形が正しく生成されること（閉鎖相が非常に短い）**
- **F0=50Hz（最低域）で位相アキュムレータの精度が維持されること（1周期=882サンプル）**
- **F0=400Hz（最高域）で1周期=110サンプルの波形が破綻しないこと**

#### 壁面損失の減衰率

- mu=0.999 適用時、44区間通過後の減衰量が理論値と一致: `0.999^44 ≈ 0.957`
- mu=1.0（無損失）で Phase 1 と同等の出力が得られること
- mu=0.0 で信号が完全に消失すること

#### 反射係数の範囲

- 全区間で |r_k| < 1 が保証されること（断面積が MIN_AREA 以上なら自動的に満たされる）
- 唇端反射 R_L が -1.0 < R_L < 0 の範囲内であること
- 声門端反射 r_glottis が 0.0 < r_glottis < 1.0 の範囲内であること

#### プリセットデータの妥当性

- 全プリセットの全制御点が MIN_AREA=0.3 以上、MAX_AREA=10.0 以下であること
- 16制御点のデータ長が正確に16であること
- /a/ の口腔部（idx 0~5）が咽頭部（idx 8~11）より広いこと
- /i/ の咽頭部（idx 7~10）が口腔前部（idx 2~4）より広いこと
- /u/ の唇端（idx 0~1）が狭窄していること
- **各母音の断面積パターンが16制御点→44区間にスプライン補間後もMIN_AREA/MAX_AREA範囲内であること**
- **neutral (Flat) の全制御点が4.0であること**
- **成功基準カバレッジ**: → REQUIREMENTS.md 8項「5母音のプリセットが正しい母音に聞こえる」のデータ正当性検証

#### コサイン補間の遷移テスト

- **t=0.0 のとき補間結果が開始値の断面積と一致すること**
- **t=1.0 のとき補間結果がターゲット値の断面積と一致すること**
- **0 < t < 1 のとき全16制御点の補間値が開始値とターゲット値の間に収まること（オーバーシュートなし）**
- **遷移中に別のプリセットを選択した場合、現在の中間状態が新しい開始値として使用されること**

#### 有声/無声クロスフェードのテスト

- **クロスフェード中のゲイン合計（AV + AN）が常に1.0以下であること（クリッピング防止）**
- **クロスフェード長（256-441サンプル）がサンプル精度で正しいこと**
- **クロスフェード開始直後の出力にDC成分（直流オフセット）が発生しないこと**

#### パフォーマンステスト — Phase 2 累積コスト
- **テスト内容**: KLGLOTT88 + 壁面損失(88乗算) + 改良放射フィルタを含むprocess()の実行時間

- 切替時にクリックノイズが発生しない（クロスフェードが機能している）
- 再度押下で有声音源に戻る

#### Phase 1 リグレッションチェック
- **Phase 1 の全ユニットテストがPhase 2コード統合後もパスすること**
- **均一管でのKLアルゴリズム理論値照合テストが壁面損失追加後も許容誤差内であること**

## 5. 実装に関する懸念事項とレビュー項目

### 5.1 KLGLOTT88 の数値安定性

**懸念**: 位相アキュムレータの浮動小数点精度。

- `phase += F0 / sampleRate` を毎サンプル繰り返すと、浮動小数点の丸め誤差が蓄積し、位相がドリフトする可能性がある
- **対策**: `phase -= 1.0` によるリセットで問題なし。ただし F0 が極端に低い場合（< 50Hz）や高い場合（> 500Hz）での動作を検証すること
- **レビュー項目**: 1時間相当（44100 * 3600 = 約1.6億サンプル）の連続実行後に位相精度が維持されるか

### 5.2 壁面損失のチューニング

<!-- REVIEW: Phase間整合性レビューにて修正 — 壁面損失(mu=0.999)はPhase 1のスコープ内で導入済み。Phase 2での懸念は「導入タイミング」ではなく「KLGLOTT88+改良放射フィルタとの組み合わせでのチューニング」に修正 -->
**懸念**: Phase 1 で導入済みの壁面損失（mu=0.999固定）が、KLGLOTT88 + 改良放射フィルタとの組み合わせで最適な値であるか。

- Phase 1 の mu=0.999 は三角波パルス + 差分放射フィルタでの値。音源と放射フィルタが変わるため再チューニングが必要な可能性がある
- **結論**: mu=0.999 を初期値として出発し、母音プリセットの聴覚評価に基づき 0.995-0.9999 の範囲で微調整する（セクション6.3補足に記載の方針と一致）
- **レビュー項目**: mu 値の変更前後で音質比較を行い、最適値を確認する

### 5.3 母音プリセットのチューニング

**懸念**: TECHNICAL_RESEARCH の断面積データは理論的な文献値ベースであり、実装後の音声が必ずしも目標フォルマントに一致するとは限らない。

- Kelly-Lochbaum の境界条件（R_L, r_glottis）、壁面損失 mu、放射フィルタ alpha がフォルマント位置に影響する
- **対策**: Phase 3 でフォルマント計算が実装されたら、計算値と目標値を比較し、プリセットデータを微調整する
- **レビュー項目**: Phase 2 の段階では聴覚的な母音識別が主な評価基準。精密なチューニングは Phase 3 以降

### 5.4 日本語 /u/ の非円唇特性

**懸念**: 日本語の /u/ は [ɯ]（非円唇後舌狭母音）であり、英語の /u/（円唇）とは異なる。

- プリセットデータでは唇端の断面積を 0.5 cm² に設定（英語の /u/ なら 0.3 cm² 程度）
- **レビュー項目**: 実際に出力される音が日本語の /u/ に聞こえるか、ネイティブ話者が確認する

### 5.5 process() 内のパフォーマンス

**懸念**: KLGLOTT88 の計算、壁面損失の追加乗算、改良放射フィルタが process() の処理時間を増加させる。

- Phase 1 の process(): 44区間 KL 散乱 + 単純パルス + 差分フィルタ
- Phase 2 の process(): 44区間 KL 散乱 + 壁面損失(88乗算追加) + KLGLOTT88(4-5 ops) + 1次IIRフィルタ
- 128 samples/block = 2.9ms の予算。44区間 KL は約 200 ops/sample、追加分を含めても余裕あり
- **レビュー項目**: Chrome DevTools の Performance タブで process() の実行時間を計測

### 5.6 postMessage の頻度

**懸念**: 遷移中の断面積更新が requestAnimationFrame（60fps = 16.7ms間隔）で postMessage を送信する。

- 128 samples = 2.9ms のレンダリングクォンタムに対し、16.7ms 間隔の postMessage は十分低頻度
- **レビュー項目**: 遷移中にオーディオグリッチが発生しないか確認

## 6. 一から作り直すとしたら

### 6.1 Phase 1 + Phase 2 の統合設計

もし Phase 1 と Phase 2 を最初から一体として設計するなら:

#### 壁面損失を Phase 1 から入れるべきだったか

<!-- REVIEW: Phase間整合性レビューにて修正 — Phase 1 のスコープ内に壁面損失(mu=0.999)が含まれているため、「入れるべきだった」ではなく「入れた判断は正しかった」に修正 -->
**結論: Phase 1 に含めた判断は正しかった**

TECHNICAL_RESEARCH 2.3節で「音質への影響大、早期実装推奨」と明記されており、CLAUDE.md にも「音質に必須、初期実装から組み込む」と記載されている。Phase 1 のスコープに壁面損失（mu=0.999固定）を含めた判断は正しい:

- Phase 1 の音質が向上し、「音が出る」マイルストーンの達成感が高まる
- Phase 2 でのデバッグ要因が1つ減る（壁面損失が問題か音源が問題か切り分けやすい）
- KL ループ内の乗算2回追加のみで実装コストが極めて低い

#### 音源モデルの抽象化レイヤー

Phase 1 のパルス音源 → Phase 2 の KLGLOTT88 → Phase 5 の LF モデルと段階的に置き換える。Phase 1 の rebuild では「Phase 1 では抽象クラス/Strategy パターンは不要、Phase 2 で差し替え時にインターフェースを抽出する」としており、Phase 5 の rebuild でも「Phase 1 の時点で GlottalSourceInterface を定義しておけばよりクリーンだった」と振り返っている。

**Phase 2 からの結論**: Phase 1 で関数シグネチャ `generate(phase: number): number` を統一しておいたのは正しい。Phase 2 では KLGLOTT88 への差し替えに合わせて、このシグネチャを正式なインターフェースとして抽出する:

```typescript
interface GlottalModel {
  /** 現在の位相（0.0~1.0）から1サンプルの声門波形値を返す */
  generate(phase: number): number;
  /** パラメータ更新 */
  setParams(params: GlottalParams): void;
}

// 各モデル固有のパラメータは判別共用体で管理
type GlottalParams =
  | { type: 'klglott88'; openQuotient: number; amplitude: number }
  | { type: 'lf'; rd: number; amplitude: number };  // Phase 5 で追加
```

Phase 1 → 2 のタイミングでの抽出は合理的（YAGNI原則）。ただしメソッド名は Phase 5 の `GlottalModel.generate()` と統一する（Phase 2 で `getSample` を導入すると Phase 5 で改名が必要になる）。パラメータ型は判別共用体にすることで、KLGLOTT88 の OQ と LF の Rd を型安全に分離できる。

#### プリセットデータの管理方法

統合設計では、プリセットデータを以下のように構造化する:

```typescript
interface VowelPreset {
  id: string;           // 'a', 'i', 'u', 'e', 'o', 'neutral'
  label: string;        // 'あ', 'い', 'う', 'え', 'お', 'Flat'
  controlPoints: number[]; // 16要素の断面積配列 (cm²)
  targetFormants?: {    // チューニング検証用（Phase 3 で活用）
    f1: number;
    f2: number;
    f3: number;
  };
}
```

`targetFormants` はオプショナルとする。Phase 3 のフォルマント計算は断面積 → 反射係数 → LPC多項式 → QR法で直接算出するため、プリセットに目標値を持たせるのは音声合成に必須ではない。ただし、TECHNICAL_RESEARCH 4.3節のフォルマント目標値をここに記録しておくことで、Phase 3 でのチューニング検証（計算値 vs 目標値の自動比較）に活用できる。neutral プリセットなど目標値が不要なケースもあるため、optional が適切。

プリセットデータは独立モジュール（`vowel-presets.ts`）に集約する。将来の子音対応に備えて `id` を文字列型（母音 'a' / 子音 'k' 等）にしておくが、Phase 2 時点では子音用フィールド（狭窄位置、ノイズ源位置など）は追加しない。

### 6.3 補足: レビューによる追加事項

#### 壁面損失パラメータの Phase 2 でのチューニング方針

Phase 1 の rebuild で「壁面損失は Phase 1 から入れるべき」と結論し、CLAUDE.md にも「初期実装から組み込む」と明記されている。Phase 2 では KLGLOTT88 + 改良放射フィルタとの組み合わせで音質が大きく変わるため、mu=0.999 を固定値として出発しつつ、母音プリセットの聴覚評価に基づき mu を 0.995〜0.9999 の範囲で微調整する工程を結合テストに含める。区間ごとの可変 mu（断面積依存）は Phase 2 では見送り、均一値で十分な音質が得られるか検証する。

#### 有声/無声切替時のクロスフェード設計

セクション 2.7 で「5〜10ms のクロスフェード」を規定しているが、rebuild 観点での設計上の注意:

- クロスフェードはサンプル単位の線形ゲインランプで実装する（コサイン補間は不要、知覚差は小さい）
- ランプ長は 256〜441 samples（5.8〜10ms at 44100Hz）を定数で定義し、process() 内で条件分岐なく処理できるよう状態変数（残りランプサンプル数）で管理する
- 切替コマンド（postMessage）が process() ブロック境界の途中で届く場合でも、次ブロックの先頭からランプを開始すれば問題ない（最大 2.9ms の追加レイテンシは許容範囲）
- Phase 5 で気息ノイズ（Aspiration）が追加されるため、クロスフェードのゲイン管理は `AV`/`AN` の2チャンネル独立で設計しておく

#### 日本語 /u/ の断面積データの妥当性

セクション 5.4 で「唇端の断面積を 0.5 cm²」としているが、定量的な検討を補足する:

- 日本語 /u/ = [ɯ]（非円唇）の唇端断面積は、MRI データ（Story et al. 1996）に基づく英語 /u/（円唇、0.2〜0.3 cm²）より明確に広い
- 0.5 cm² は MIN_AREA=0.3 cm² の制約を満たしつつ、非円唇特性を反映する妥当な値。ただし実際には 0.5〜1.0 cm² の範囲が考えられ、Phase 3 でフォルマント計算が利用可能になった時点で F1/F2 目標値（350/1400 Hz）との照合による微調整が必要
- F2 が 1400 Hz（英語の /u/ の約 800〜1000 Hz より高い）であることが日本語 /u/ の最大の特徴であり、これは舌位置（idx 8〜10 の狭窄）で主に制御される。唇端の断面積は F2 への影響が相対的に小さいため、0.5 cm² からの調整幅は限定的と予想される

#### コサイン補間による母音遷移の通信レイテンシ評価

セクション 2.8 の 80〜200ms コサイン補間は requestAnimationFrame（16.7ms 間隔）で postMessage を送信する設計。遷移中に約 5〜12 フレームの更新が入る計算になる。postMessage の到着遅延は通常 1ms 以下であり、16.7ms の更新間隔に対して無視できる。80ms の最短遷移でも約 5 フレームの補間ステップが確保されるため、知覚上は十分滑らか。ただし、requestAnimationFrame がバックグラウンドタブでスロットルされる点に注意し、`document.hidden` の場合は setInterval（4ms 精度）へのフォールバックを検討する。

### 7.3 Phase 4（自動歌唱モード）へ

#### プリセット遷移APIの仕様

Phase 2 で実装する遷移マネージャを Phase 4 の Auto Sing モードから利用するため、以下の API を公開する:

```typescript
interface TransitionManager {
  /**
   * 指定した母音プリセットへの遷移を開始する
   * @param targetPresetId - 'a' | 'i' | 'u' | 'e' | 'o' | 'neutral'
   * @param durationMs - 遷移時間（ms）。デフォルト 150ms
   */
  transitionTo(targetPresetId: string, durationMs?: number): void;

  /**
   * 任意の断面積配列への遷移を開始する（プリセット以外の形状へ）
   * @param targetControlPoints - 16要素の断面積配列
   * @param durationMs - 遷移時間（ms）

  transitionToCustom(targetControlPoints: number[], durationMs?: number): void;

  /** 遷移中かどうか */
  isTransitioning(): boolean;

  /** 現在の16制御点の断面積値を取得 */
  getCurrentControlPoints(): number[];
}
```

Phase 4 の Auto Sing モードでは:
- `transitionTo()` で母音遷移を実行
- `durationMs` を 80~200ms の範囲で制御（母音間の F1-F2 距離に比例）
- `isTransitioning()` で遷移完了を待機してから次の母音を選択
